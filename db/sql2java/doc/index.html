Running the SQL2JAVA example

The SQL2JAVA example can be run in less than five minutes. It generates a DB layer API to access a sample hsql database schema, Javadoc, and more!

We assume Java and Ant have already been installed.

  1. First, start the hsql database. From a console, type: ant hsql.server
    NOTE: In Linux, type ant hsql.server & to keep hsql running in the background and immediately regain access to the console. If you accidentally forget the &, you can still enter ctrl+z, bg to move hsql.server to the background.
    NOTE: In Windows, type START ant hsql.server to keep hsql running in the background and immediately access a new command console.)
  2. Create the sample database: ant database.create
  3. Generate the Java API for accessing the sample database: ant generate, the generated code was placed here...
  4. Compile the generated source code API: ant compile
  5. Run the Main.java program which uses the sample source code API: ant run
  6. Type ant javadoc to generate the javadoc
  7. Try sql2java against your own schema. Take a look at sql2java.properties; it is self documented.
  8. To customize the generated code, edit sql2java velocity templates:
    • database.properties,
    • GeneratedBean.java,
    • Manager.java,
    • xBean.java,
    • xComparator.java,
    • xListener.java,
    • xManager.java
  9. You can also add your own templates here ...
  10. Enjoy!
